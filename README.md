# CostPilot

프로젝트 원가·수익성 분석 툴입니다. 인력 투입, 직접비, 외주비, 공통비를 통합해 프로젝트별 손익과 원가 동인을 분석합니다.

## 주요 기능

- 프로젝트별 손익 계산: 매출, 내부 인건비, 외주 용역비, 직접경비, 배부 공통비, 총원가, 순이익, 이익률
- 경영 대시보드: 원가율, 저마진 프로젝트, 외주 의존 리스크, 본부별 성과 스냅샷
- 공통비 배부 엔진: 투입 시간, 인건비, 매출 기준 배부
- 본부별 공통비 풀: 전사 공통비와 특정 본부 공통비 분리
- 프로젝트별 배부 가중치
- What-if 시뮬레이션: 인력 추가, 매출 증감, 공통비 증감, 이익 변화 폭포수 차트
- 시나리오 저장/적용/삭제
- 데이터 관리: 인력 마스터, 프로젝트 마스터, 일일 근무 기록, 비용 원장 추가/삭제/수정
- 사용자 데이터 가져오기: 엑셀 템플릿 다운로드, XLSX 업로드 검증, 오류/경고 검토 후 전체 교체 반영
- 엑셀 기반 데이터 가져오기/내보내기
- 경영 인사이트 리포트: 수익성 상위, 개선 후보, 외주 의존도, M/M 효율, 공통비 민감도

## Vercel 배포

- Framework Preset: `Vite`
- Install Command: `npm install` 또는 `npm ci`
- Build Command: `npm run build`
- Output Directory: `dist`

`vercel.json`에는 SPA 라우팅을 위한 rewrite 설정이 포함되어 있습니다.

배포 전 로컬 체크:

```bash
npm run test
npm run build
npm audit --omit=dev
```

현재 앱은 별도 서버/API/DB가 필요하지 않습니다. 데이터는 브라우저 localStorage에 저장되며, Vercel 정적 배포만으로 동작합니다.

## 시연 흐름

1. `대시보드`에서 전체 매출, 총원가, 순이익, 이익률을 확인합니다.
2. `데이터 관리`에서 엑셀 템플릿을 내려받고 사용자 데이터를 업로드합니다.
3. `배부 엔진`에서 투입 시간/인건비/매출 배부 기준별 결과를 비교합니다.
4. `시뮬레이션`에서 특정 프로젝트에 인력을 추가하거나 매출/공통비 변화를 입력합니다.
5. 현재 시나리오를 저장하고 여러 시나리오의 전체 순이익 영향을 비교합니다.
6. `리포트`에서 경영 인사이트, 본부별 수익성, 수익성/리스크 프로젝트를 확인하고 PDF 또는 엑셀로 내보냅니다.

## 엑셀 Import 양식

`데이터 관리 > 엑셀 연동 > 엑셀 템플릿`에서 XLSX 파일을 다운로드합니다. 템플릿은 다음 시트로 구성됩니다.

- `작성 안내`
- `인력 마스터`
- `프로젝트 마스터`
- `프로젝트 투입`
- `일일 근무 기록`
- `비용 원장`

사용자는 내부 ID를 입력하지 않고 `사번`, `프로젝트코드`, `본부명` 기준으로 데이터를 작성합니다. 업로드 시 앱이 내부 ID를 자동 생성하고 참조 관계를 검증합니다.

주요 규칙:

- `판관비`는 전사 공통비로 처리되므로 프로젝트코드를 비웁니다.
- `직접경비`와 `외주비`는 프로젝트코드가 필요합니다.
- 일일 근무 기록의 사번/프로젝트코드는 마스터 시트에 먼저 존재해야 합니다.
- 오류가 있으면 반영할 수 없고, 경고만 있으면 확인 후 반영할 수 있습니다.

## 데이터 저장 방식

별도 DB 없이 브라우저 localStorage에 저장합니다. 샘플 데이터로 되돌리고 싶으면 `데이터 관리` 탭의 `샘플 데이터 초기화`를 사용하면 됩니다.
