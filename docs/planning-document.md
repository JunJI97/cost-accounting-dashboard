# CostPilot 기획서

## 1. 프로젝트 개요

CostPilot은 여러 본부가 동시에 수행하는 프로젝트의 매출, 인건비, 외주비, 직접경비, 공통비를 통합하여 프로젝트별 수익성과 원가 동인을 분석하는 관리회계 웹 애플리케이션이다. 기존 엑셀 중심 원가 관리에서는 공통비 배부 기준이 불명확하거나 프로젝트별 손익 변동을 즉시 확인하기 어렵다는 문제가 있다. 본 프로젝트는 이러한 문제를 해결하기 위해 데이터 입력, 원가 배부, 수익성 분석, 시뮬레이션, 리포트 생성을 하나의 화면 흐름으로 제공한다.

## 2. 기획 배경

SI, 컨설팅, 개발 용역과 같이 인력 투입 비중이 높은 조직에서는 프로젝트별 수익성을 정확히 계산하기 위해 인력 투입 시간과 표준 인건비를 함께 관리해야 한다. 또한 사무실 임대료, 관리 인력 급여, 공용 SaaS 비용처럼 특정 프로젝트에 직접 귀속하기 어려운 공통비를 어떤 기준으로 배부하느냐에 따라 프로젝트별 이익률과 개선 우선순위가 달라진다.

CostPilot은 단순 비용 합산 도구가 아니라, 배부 기준을 바꿔 보며 손익 변화를 비교하고 추가 인력 투입이나 매출 변화가 전체 수익성에 미치는 영향을 검토할 수 있는 의사결정 도구를 목표로 한다.

## 3. 목표 사용자

- 프로젝트별 원가와 수익성을 관리해야 하는 경영관리/관리회계 담당자
- 본부별 프로젝트 성과를 비교해야 하는 임원 또는 팀장
- 인력 투입 계획과 외주비 사용을 점검해야 하는 PM
- 과제 또는 실습 환경에서 원가 배부 로직과 프론트엔드 구현을 함께 확인해야 하는 평가자

## 4. 주요 문제 정의

1. 프로젝트별 인건비, 외주비, 직접경비, 공통비가 흩어져 있어 실제 총원가를 한눈에 보기 어렵다.
2. 공통비 배부 기준이 바뀌면 프로젝트별 순이익과 이익률이 달라지지만 이를 즉시 비교하기 어렵다.
3. 추가 인력 투입, 매출 증감, 공통비 증감 같은 의사결정이 손익에 미치는 영향을 사전에 확인하기 어렵다.
4. 현업 데이터는 엑셀 기반으로 관리되는 경우가 많아 앱으로 가져오기 위한 검증 절차가 필요하다.
5. 분석 결과를 제출 또는 보고용 리포트로 정리하는 과정이 별도로 필요하다.

## 5. 핵심 기능

### 5.1 대시보드

- 전체 매출, 총원가, 순이익, 평균 이익률 KPI 표시
- 프로젝트별 수익성 테이블 제공
- 본부별 매출/순이익 비교 차트 제공
- 비용 구성 비중과 원가 구성 Top 8 시각화
- 저마진 프로젝트, 손실 프로젝트, 외주 의존도가 높은 프로젝트를 빠르게 식별

### 5.2 데이터 관리

- 인건비, 프로젝트, Daily Report, 직접비/공통비 데이터를 탭 단위로 관리
- 사번, 프로젝트코드, 본부명 등 업무 키 기반 데이터 편집
- 샘플 데이터 초기화 기능 제공
- 변경 데이터는 브라우저 localStorage에 저장되어 새로고침 후에도 유지

### 5.3 엑셀 가져오기/내보내기

- 통합 엑셀 템플릿 다운로드
- 인력 마스터, 프로젝트 마스터, 프로젝트 투입, 일일 근무 기록, 비용 원장 시트 업로드
- 필수값 누락, 참조 오류, 숫자 형식 오류, 중복 오류 검증
- 경고와 오류를 구분하여 오류가 없을 때 전체 데이터 교체 반영
- 현재 데이터셋 또는 일부 데이터 시트를 엑셀 파일로 내보내기

### 5.4 공통비 배부 엔진

- 투입 시간 기준 배부
- 인건비 기준 배부
- 매출 기준 배부
- 프로젝트별 배부 가중치 반영
- 전사 공통비와 본부별 공통비 풀을 구분할 수 있는 구조 제공
- 기준별 프로젝트 손익 변화와 최대 변동 프로젝트 비교

### 5.5 What-if 시뮬레이션

- 특정 프로젝트에 추가 인력 투입
- 인당 투입 시간과 시급 조정
- 프로젝트 매출 증감 입력
- 공통비 증감 입력
- 현재 손익과 변경 후 손익 비교
- 수익 변화 워터폴 차트와 영향도 Top 5 표시
- 시나리오 저장, 적용, 삭제

### 5.6 경영 인사이트 리포트

- 전체 이익률, 외주비 비중, 내부 인건비 비중, 손실 프로젝트 수 요약
- 수익성 상위 프로젝트와 개선 후보 프로젝트 표시
- 외주 의존도, M/M 효율, 공통비 민감도 분석
- 본부별 수익성 테이블 제공
- PDF 저장 및 리포트 엑셀 다운로드

## 6. 데이터 구성

| 구분 | 주요 필드 | 설명 |
| --- | --- | --- |
| 본부 | id, name | 프로젝트와 인력의 소속 조직 |
| 인력 | 사번, 이름, 본부, 시급, 기본급, 수당 | 인건비 계산 기준 |
| 프로젝트 | 프로젝트코드, 프로젝트명, 본부, 매출, 기간, 배부 가중치 | 수익성 분석 대상 |

| 비용 원장 | 지출일, 프로젝트코드, 비용구분, 금액, 내역 | 직접경비, 외주비, 판관비 원천 데이터 |

## 7. 핵심 계산 방식

프로젝트별 내부 인건비는 직원의 시간당 표준원가와 프로젝트 투입 시간을 곱해 계산한다. 프로젝트에 직접 연결된 비용 중 직접경비와 외주비는 해당 프로젝트의 직접비로 집계한다. 프로젝트코드가 없는 판관비 또는 공통비는 선택한 배부 기준에 따라 프로젝트별로 나누어 배부한다.

최종 프로젝트 손익은 다음과 같이 계산한다.

```text
총원가 = 내부 인건비 + 외주비 + 직접경비 + 배부 공통비
순이익 = 매출 - 총원가
이익률 = 순이익 / 매출
M/M = 투입시간 / 160
```

## 8. 화면 구성

| 화면 | 목적 | 주요 기능 |
| --- | --- | --- |
| 대시보드 | 전체 현황 파악 | KPI, 차트, 프로젝트 수익성 테이블 |
| 데이터 관리 | 원천 데이터 관리 | 인력/프로젝트/근무/비용 편집, 엑셀 연동 |
| 배부 엔진 | 공통비 기준 비교 | 투입 시간/인건비/매출 기준 비교 |
| 시뮬레이션 | 의사결정 영향 분석 | 인력, 매출, 공통비 변경 시뮬레이션 |
| 리포트 | 제출/보고 자료 생성 | 인사이트, 순위, 본부별 분석, PDF/엑셀 저장 |

## 9. 기대 효과

- 프로젝트별 수익성과 원가 구조를 한 화면에서 빠르게 확인할 수 있다.
- 공통비 배부 기준 변경에 따른 손익 변동을 투명하게 비교할 수 있다.
- 추가 인력 투입이나 매출 변화의 영향을 사전에 검토할 수 있다.
- 엑셀 기반 현업 데이터를 템플릿과 검증 절차로 안정적으로 반영할 수 있다.
- 분석 결과를 PDF 또는 엑셀 리포트로 내보내 과제 제출 및 보고에 활용할 수 있다.

- 사용자 권한 관리와 업로드 이력 관리 추가
- 부분 병합 업로드 기능 추가
- 배부 기준을 사용자가 직접 수식으로 정의하는 커스텀 배부 기능 추가
- 기간별 월간 손익 추이와 프로젝트 예산 대비 실적 분석 추가
